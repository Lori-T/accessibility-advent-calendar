Keyboard accessibility ensures that everyone, including users who rely on assistive technologies or cannot use a mouse, can interact with your website or application. This is vital for people with motor impairments, people who use screen readers, have temporary injuries, or those who simply prefer using a keyboard.

Why keyboard accessibility matters

Websites and applications should be fully navigable using only a keyboard. This allows users to perform all essential actions, like filling out forms, navigating menus, and submitting information, without relying on a mouse or touchpad.

Common keyboard accessibility issues

  • Unreachable elements,if interactive components like buttons, links, and form fields are not focusable or operable using a keyboard then they can't be used by everyone
  • Improper focus order, if focus moves in an illogical or confusing order, it makes it difficult for users to navigate
  • No visual focus indicator, if keyboard users can't see which element is currently focused, it can leave them unsure of their position on the page

Tips for improving keyboard accessibility

  • Make all interactive elements focusable, use semantic HTML for elements like button and a. For custom components, ensure they are keyboard operable using tabindex and ARIA attributes where necessary.
  • Ensure a logical focus order, check that the tab order follows the natural flow of the page. Avoid disrupting the userâ€™s navigation experience with unexpected jumps.
  • Provide a visible focus indicator, by using CSS to style the :focus state, visit the GOV.UK design system page on focus states to find out more about implementing focus states.
  • Enable keyboard navigation for complex widgets, components like modals, dropdowns, and carousels should support keyboard actions, such as arrow keys and Escape to close.

Keyboard commands reference

The table following table outlines common keyboard commands that should be supported for accessibility. This is not an exhaustive list.

Keyboard commands
Key(s) Function
Tab Move to the next focusable element
Shift + Tab Move to the previous focusable element
Enter Activate a focused link, button, or other control, select a list item (click function)
Space Open a drop down menu, activate a focused button, check/uncheck a checkbox, page down
Escape Close, menus, or dialogs, cancel/close a prompt/modal
Arrow Keys Navigate within menus, move between list items, tabs, radio buttons, scroll page
Command + L/ Ctrl + L Go to address bar

Testing keyboard accessibility

You can test your website by unplugging your mouse and navigating entirely with the keyboard. The following table provides an overview of essential keyboard interactions to check during testing:

Keyboard navigation checklist
Action Expected behaviour
Navigate between elements Use Tab to move forward and Shift + Tab to move backward, the focus should move logically through the page.
Activate links and buttons Press Enter or Space to activate interactive elements
Use dropdowns or menus Arrow keys move through options, and Enter selects an item.
Close modals or menus Press Escape to close overlays or modals.
Ensuring full keyboard support makes your site usable for a wider audience and is a key part of ensuring accessibility of a website or service.
